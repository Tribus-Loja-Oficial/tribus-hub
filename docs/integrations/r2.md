# Cloudflare R2 — Storage de assets

O tribus-hub usa **Cloudflare R2** como storage de objetos para todos os assets (imagens, PDFs, vídeos, etc.). R2 é compatível com a API S3 da AWS.

---

## Configuração

| Variável de ambiente | Descrição |
|---------------------|-----------|
| `R2_ACCOUNT_ID` | ID da conta Cloudflare |
| `R2_ACCESS_KEY_ID` | Access key com permissão de leitura/escrita no bucket |
| `R2_SECRET_ACCESS_KEY` | Secret key correspondente |
| `R2_BUCKET_NAME` | Nome do bucket (default: `tribus-hub-assets`) |
| `R2_PUBLIC_URL` | URL pública do CDN (opcional; se ausente, usa signed URLs) |

As credenciais são lidas **exclusivamente** por `src/lib/config/env.ts`.

---

## Abstração no projeto

Todo acesso ao R2 passa por `src/lib/integrations/r2/`. Nunca usar o SDK S3 diretamente em routes ou services.

```
src/lib/integrations/r2/
  client.ts     — instância do S3Client configurada
  upload.ts     — função de upload
  delete.ts     — função de deleção
  signed-url.ts — geração de signed URLs
```

---

## Formato dos object keys

```
uploads/YYYY/MM/[asset-id].[ext]
```

Exemplos:
- `uploads/2025/06/clx123abc.jpg`
- `uploads/2025/06/clx456def.pdf`

O ID do asset é gerado com `generateId()` de `src/lib/utils/ids.ts` (CUID2).

---

## Fluxo de upload

1. Client faz `POST /api/assets/upload` com o arquivo (multipart/form-data)
2. API valida mime type e tamanho no servidor (via env vars)
3. `asset.service` gera `assetId` e chama `r2.integration.upload()`
4. Upload para R2 com `Content-Type` explícito e checksum SHA-256
5. Metadados persistidos na tabela `assets` do banco
6. Retorna `{ id, url }` ao client

---

## Acesso aos arquivos

- **Arquivos públicos:** se `R2_PUBLIC_URL` estiver configurado, a URL é montada diretamente (`${R2_PUBLIC_URL}/${key}`).
- **Arquivos privados:** signed URL gerada on-demand com validade de **1 hora**. Nunca armazenar signed URLs no banco.

---

## Deleção

Ao deletar um asset:
1. Remover o objeto do R2 (`r2.integration.delete(key)`)
2. Marcar o registro no banco com `deletedAt = now()`

Nenhum objeto deve ficar órfão no storage.

---

## Tipos de uso (asset kinds)

| Kind | Uso |
|------|-----|
| `cover` | Imagem de capa de página ou projeto |
| `inline` | Imagem inserida no conteúdo do editor |
| `attachment` | Arquivo anexado a uma página ou tarefa |
| `reference` | Asset referenciado externamente |
| `avatar` | Foto de perfil de usuário |

---

## Segurança

- Mime type e tamanho validados no servidor — nunca confiar no client.
- Credenciais com escopo mínimo: leitura/escrita apenas no bucket específico.
- Signed URLs para arquivos privados; nunca expor a URL interna do bucket.

Consulte [conventions/security](../conventions/security.md) para regras completas de segurança com R2.
